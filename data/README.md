# 主页数据配置指南 (Data Configuration Guide)

这篇文档说明了主页如何通过 `data/` 目录下的各个 JSON 文件来进行内容动态渲染。你只需要修改这些 JSON 文件的值，保存后页面就会自动加载最新内容，无需修改任何代码。

---

## 1. 基础资料配置 (`profile.json`)
管理网站内绝大多数的通用基础个人信息，包含首屏自我介绍、背景故事和页脚。

- **`hero`**: 网页首屏的核心展示资料
  - `name`: 你的大号中文/英文全名
  - `title`: 你的头衔或口号（目前实际上在 HTML 中去掉了此占位符显示，但也保留了字段以备不时之需）
  - `intro`: 首屏位于名字下方的简短自我介绍（支持一小段核心导语）
- **`about`**: 对应网页内 `About` 区块的长篇个人介绍
  - 数据格式为：`["第一段文字", "第二段文字", ...]` 
  - 每一项都会在页面中生成一个单独的 `<p>` 段落
- **`footer`**: 页面底边栏
  - `name`: 页脚附带的版权所属人名称，如年份旁边的 "Your Name"

---

## 2. 精选项目配置 (`projects.json`)
管理 `Projects` 模块的项目列表。

- **`projects`**: 项目数组，按顺序在页面渲染
  - `tag`: 项目类型或者序号标识（例如：`"01 / Product UX"`）
  - `title`: 你的项目名称
  - `description`: 描述该项目到底解决了什么问题或采用了什么技术
  - `linkText`: 引导按钮文字（例如：`"查看演示"`, `"Request walkthrough"`）
  - `linkHref`: 按钮跳转链接（如果要跳转外部网站，请填写类似 `https://github.com/...`，如果仅作展示可以填 `#contact` 跳回锚点）

---

## 3. 经验轨迹配置 (`experience.json`)
管理 `Experience` 模块的职场 / 学历时间线列表。

- **`experience`**: 你的履历数组，一般建议把最新的经历放在最前面
  - `role`: 你的职位名称（例如：`"前端开发工程师"`）
  - `company`: 你所在的公司或学校（例如：`"XX 科技"`）
  - `period`: 该段经历的时长/跨度（例如：`"2022 - 2024"`）
  - `description`: 对你的职责产出、成就的简述

---

## 4. 联系社交配置 (`contact.json`)
管理右下角 `Contact` 的所有社交网络按钮及邮件连接。

- **`contact`**: 具体的联系方式按钮数组
  - `text`: 按钮显示的文本（如：`"Email Me"`, `"GitHub"`）
  - `href`: 按钮连接到的目标地址（如果要指定邮件，可以写 `mailto:你真实的邮箱地址@xxx.com`）
  - `primary`: 设置为 `true` 的按键将被渲染为主色高亮色调；设为 `false` 则为次要色调的线框按钮
  - `blank`: （可选）如果想在新窗口打开链接需要添加此项并设置为 `true`

---

## 5. 兴趣动态配置 (`interests.json`)
这是整个网页动效最足的 `Interests` 板块数据集合，聚合了三大兴趣领域的配置：

### 5.1 旅行地图 (travel)
- **`cities`**: 按顺序将连线的城市数组
  - `name`: 城市名称
  - `x`: 横向坐标（0-100间的百分比，控制该点在宽度上的位置。由于 SVG 画布的比例不同，需要你根据视觉效果稍作调试）
  - `y`: 纵向坐标（0-100间的百分比，控制高度位置）

### 5.2 摄影底片 (photography)
- **`photos`**: 呈现为一个四宫格复古相册的轮播展示数组
  - `src`: 照片文件的相对路径，这里存放的图片请放置在 `./data/photos/` 文件夹下
  - `caption`: 这张照片悬浮时下方弹出的旁白介绍文字

### 5.3 音乐集 (music)
- **`songs`**: 当用户每次打开该组件时，它会按顺序**轮换**抽取播放数组中的其中一首歌词进行汇聚展示
  - `title`: 歌手名和歌曲名
  - `lyrics`: 纯文本数组。最好在 6~8 句短句之间，每项会在屏幕四周生成并逐渐浮向屏幕中央组装成完整诗句
